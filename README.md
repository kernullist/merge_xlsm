# Macro VBA XLSM Merge

이 프로젝트는 두 개의 매크로가 포함된 엑셀 파일(`.xlsm`)을 하나로 병합하는 파이썬 스크립트입니다.
단순히 워크시트를 복사하는 것을 넘어, **VBA 매크로 모듈(Standard, Class, UserForm)**까지 함께 병합합니다.

이 스크립트는 엑셀의 COM(Component Object Model) 인터페이스를 사용하여 백그라운드에서 엑셀을 직접 제어하므로, **반드시 엑셀이 설치된 Windows 환경**에서 실행해야 합니다.

## 사전 준비 (Prerequisites)

- Windows 운영체제
- Microsoft Excel 설치됨
- Python 3.x

## 설치 방법 (Installation)

이 스크립트는 엑셀을 제어하기 위해 `pywin32` 라이브러리를 사용합니다. 아래 명령어로 필요한 패키지를 설치해 주세요.

```bash
pip install -r requirements.txt
```

## ⚠️ 필수: 엑셀 매크로 보안 설정

VBA 코드(매크로)를 병합하려면, 외부 프로그램이 엑셀의 VBA 프로젝트에 접근할 수 있도록 권한을 허용해야 합니다. 이 설정을 하지 않으면 시트만 병합되고 VBA 모듈은 병합할 때 오류가 발생하거나 누락될 수 있습니다.

1. **Excel** 프로그램을 실행합니다.
2. 상단 메뉴에서 **파일** > **옵션**을 클릭합니다.
3. 왼쪽 메뉴에서 **보안 센터**를 선택하고, **보안 센터 설정** 버튼을 클릭합니다.
4. 왼쪽 메뉴에서 **매크로 설정**을 선택합니다.
5. 개발자 매크로 설정 항목에서 **"VBA 프로젝트 개체 모델에 안전하게 액세스할 수 있음"** (Trust access to the VBA project object model) 항목을 찾아 **체크**합니다.
6. 확인을 눌러 설정을 저장하고 엑셀을 종료합니다.

## 사용 방법 (Usage)

터미널이나 명령 프롬프트에서 아래와 같이 스크립트를 실행합니다.

```bash
python merge_xlsm.py <원본1.xlsm> <원본2.xlsm> <결과물.xlsm>
```

**예시:**
```bash
python merge_xlsm.py data1.xlsm data2.xlsm merged_result.xlsm
```

## 동작 원리

1. 스크립트가 백그라운드에서 엑셀을 실행(화면에는 보이지 않음)하여 지정된 두 원본 파일을 엽니다.
2. 두 번째 파일(`원본2`)의 모든 워크시트를 첫 번째 파일(`원본1`)의 마지막 시트 뒤로 복사합니다.
3. 두 번째 파일의 VBA 컴포넌트(일반 모듈, 클래스 모듈, 사용자 정의 폼)를 시스템 임시 폴더로 내보낸(Export) 후, 첫 번째 파일로 가져옵니다(Import).
4. 병합이 완료된 상태를 지정한 `<결과물.xlsm>` 이름으로 저장하고, 열려있던 파일들을 닫은 뒤 엑셀을 안전하게 종료합니다.

## 주의 사항

- 병합하려는 두 파일에 **동일한 이름의 VBA 모듈이나 시트**가 존재할 경우, 이름이 자동 변경되거나 충돌(오류)이 발생할 수 있습니다. 가급적 병합 전 모듈 이름이 겹치지 않게 구성하는 것이 좋습니다.
- 스크립트 실행 중 강제 종료 등의 이유로 비정상 종료된 경우, 백그라운드에 `EXCEL.EXE` 프로세스가 남아있을 수 있습니다. 이때는 작업 관리자(Ctrl+Shift+Esc)에서 수동으로 엑셀 프로세스를 종료해 주어야 합니다.
